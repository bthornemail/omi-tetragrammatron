Core Realization

The earlier name Omi-Talisman was useful because it suggested a personal protective encoding: a mnemonic sigil, a carrier of meaning, a protective amulet, a resonance object.
But the deeper technical name is:
Omi-Ring
A ring is better because it means:
enclosure
cycle
return
orbit
boundary

agreement loop
The Omi-Ring is not merely something worn or carried.
It is the atomic bounded waveform of an OMI object.
It is the minimal enclosure where intent can be preserved without yet being fully interpreted.
Canonical line:
Omi-Ring is the atomic waveform enclosure of omi---imo.

2. Public View and Introspective View
The podcast reading was right:
Omi-Portal = public view
Omi-Mirror = introspective view
An Omi-Portal shows how an object becomes publicly inspectable.
It is outward-facing.
It asks:
How does this agreement appear to others?
How can this object be entered, viewed, replayed, or shared?
An Omi-Mirror shows how an object reflects back into itself.
It is inward-facing.
It asks:
How am I being seen?
What is missing from the projection?
What does this reflection reveal?
What does it omit?
So:
Omi-Portal projects outward.
Omi-Mirror reflects inward.
Omi-Ring encloses the waveform between them.
The portal and mirror are views.
The ring is the bounded unit being viewed.

12. Why IEEE 754 Helps as an Analogy
IEEE 754 is useful for a different reason.
It teaches that number representation is not the same as number itself.
A floating-point value has:
sign
exponent
mantissa
rounding behavior
special values
precision limits
That helps explain why OMI avoids making raw projection the authority.
A rendered avatar, waveform, glyph, or light trace is like a representation.
It has precision limits.
It has encoding assumptions.
It has rounding.
It has edge cases.
OMI therefore keeps the frame separate from the projection.
Canonical line:
Representation is not authority; representation is a recoverable view under rules.
